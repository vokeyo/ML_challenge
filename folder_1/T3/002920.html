Cómo Bajar una Tabla Rápida

Hay varios formatos de archivos disponibles para bajar las Tabla Rápida: Tres formatos listos para presentación y dos formatos compatibles con bases de datos

El formato de presentación conserva el formato de tabla, el título, nota titular y nota(s) al calce exactamente como se muestran en la pantalla.
  • Delimitado por comas (.csv): las celdas de datos están separadas por comas. Este formato es compatible con programas de hoja de cálculo como Microsoft Excel.
  • Delimitado por tabulaciones (.lst): las celdas de datos están separadas por tabulaciones.
  • Formato Rich text file - este es el formato compatible con muchos programas de procesamiento de palabras Si su explorador reconoce el formato .rtf abrirá el archivo automáticamente en la misma ventana. Utilice Archivo | Salvar Como del menú para guardar el archivo en su computadora.
  • Microsoft Excel (.xls) - este es el formato de archivo compatible con Microsoft Excel y muchos otros programas de hojas de cálculos. Fíjese que este formato no está disponible (la opción está sombreada) si la combinación de tablas y selecciones geográficas suman a más de 25 tablas.
El límite de la selección de Tabla Rápida es 10; el límite de la selección geográfica es 7000. Si selecciona 10 tablas y 100 geografías tendrá 1000 tablas individuales. Si hay más de 25 tablas individuales, aparece la opción muestre sólo las tablas y geografías en la pantalla. Si marca esta opción, sólo se bajarán las tablas y geografías que aparecen en la pantalla. Esta opción sólo aplica a los formatos de presentación listados anteriormente.

Los formatos de bases de datos, sólo se bajan filas de datos. El formato de tabla, el título, nota titular o nota(s) al calce se excluyen al bajar el archivo. Utilice estos formatos si va a bajar los datos a un programa de base de datos.

  • Microsoft Excel (.xls) - este es el formato de archivo compatible con programas de hojas de cálculos,
  • Base de datos delimitados por comas (.txt )- las celdas de datos están separadas por comas.
Los archivos bajados compatibles con bases de datos se comprimen en un solo archivo titulado output.zip. Este archivo comprimido contiene:

  • Un o más archivos de datos - El formato del nombre y el número de estos archivos variará por conjunto de dato y/o por el número de tablas que haya seleccionado.
  • Un archivo de identificador geográfico - este archivo le permite enlazar múltiples archivos de datos.
  • Un archivo gct_readme.txt - este archivo explica el contenido de los archivos bajados, cómo enlazar los archivos de datos, anomalías específicas de los conjuntos de datos y la localización de la documentación técnica específica a los conjuntos de datos.
  • Un archivo readme.txt para un conjunto de datos específico - este archivo explica por qué puede que no haya datos para algunas selecciones geográficas.
Puede incluir el nombre descriptivo largo marcando el encasillado de cotejo incluya nombres de elementos de datos descriptivos. Cada elemento de datos para cada tabla tiene un nombre corto en clave y un nombre descriptivo largo. Por ejemplo:

  • Nombre corto en clave: P001001 donde P001 es el número de tabla y 001 es el número de elementos de datos en esa tabla.
  • Nombre descriptivo largo: Población Total: Total

CONSEJOS para usar el formato compatible con bases de datos:

  • Puede guardar output.zip con ese nombre o con otro de su creación.
  • Asegúrese de consultar el archivo readme.txt para explicación de todos los tipos de archivos que ha bajado y como enlazarlos
